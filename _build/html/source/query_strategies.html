query_strategies.strategy module¶

class query_strategies.strategy.Strategy(X, Y, idxs_lb, net, handler, args)[source]¶

Bases: object

The base class wrapping the training and querying procedure. All new strategies should be inherited from this class.

Typically in a new strategy, we can just overload the query member function, which returns the indices of queried samples.

Args:
X: (Any)

Any form of raw data. It should be the train split of the dataset.

Y: (Any)

The labels of X.

idxs_lb: np.ndarray(bool)

An boolean array that has the same length with the train split of the dataset.

net: (class)

A net wrapper, with a __call__ member that returns the model architecture, its parameters are just the parameters of the model class, arg num_classes is necessary.

handler: (torch.utils.data.Dataset)

Dataset wrappers designed for active learning, whose __getitem__ returns (data, label, idx)

args: (dict)

Contains lots of information the strategy needs for training, including the ones in the parser and some newly added ones. Here we list some of them for detailed explanation.

  • work_dir: (str) The working directory, in which we save checkpoints, log info, figures

  • save_freq: (int) Save a checkpoint per save_freq epochs.

  • n_epoch: (int) How many epochs are in one active round.

  • batch_size: (int) Just the batch size of the dataloader.

  • num_workers: (int) num_workers of the dataloader.

  • lr: (float) Learning rate of the optimizer.

  • momentum: (float) Momentum of the optimizer.

  • out_iter_freq: (int) Print a log message per out_iter_freq iters.

  • num_classes: (int) The number of classes in the dataset.

  • label2text: (dict[int, str]) The class name corresponding to each label.

  • train_transform: (torchvision.transforms.Compose)Transformation applied to the training data.

  • eval_transform: (torchvision.transforms.Compose)Transformation applied to the eval/test data.

  • timestamp: (str) The start timestamp of the process.

  • logger: (logging.RootLogger) The logger to use.
